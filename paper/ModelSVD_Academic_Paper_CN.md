## 摘要

本文提出 ModelSVD，一种面向大语言模型的领域知识提取、编码、迁移与部署的综合框架，全程无需梯度计算。该框架包含两条互补的处理流水线：(1) 权重提取注入，通过奇异值分解 (SVD) 对训练后的权重增量进行分解，以分离承载知识的奇异方向；(2) 文本到 SVD 编码，通过隐藏状态回归和 Dagger 伪逆投影，直接从文本文档编码知识。两条流水线均无需反向传播。我们进一步提出 KARL HBC（知识增强推理层与超网络基向量组合），一种部署架构，通过共享基向量表示将等效 50B 参数的知识容量压缩至 1.68B 活跃参数，实现 30 倍压缩比。基向量权重通过 Hebbian 学习配合 Oja 规则稳定化进行更新，完全消除了基于梯度的优化。九个加速模块通过流式随机化 SVD、草图化 Hebbian 更新和随机层注入等技术，提供额外 10-100 倍的训练加速。在 Qwen2.5-Coder-1.5B-Instruct 上的实验表明，SVD 编码注入在 HumanEval 上达到 55.5%（基线 15.9%，提升 +39.6pp），在 GSM8K 上达到 60.0%（基线 38.3%，提升 +21.7pp），注入时间仅为 28-32 秒。该方法展现出确定性可复现性（30 次独立试验 sigma = 0.0000%）、相比等效 LoRA 训练 19 倍的端到端加速，以及正向跨域溢出效应。ModelSVD 建立了一种新范式，其中知识是一等公民——可提取、可净化、可迁移、可独立于模型训练进行部署。

## 1. 引言

赋予大语言模型 (LLM) 领域特定能力的主流范式是基于梯度的训练——无论是全量微调、参数高效方法如 LoRA [1]，还是知识蒸馏 [2]。这些方法共享以下根本性局限：

1. **计算成本**：训练一个 1.5B 参数的模型需要每个领域数个 GPU 小时的计算量，且随模型规模和数据量线性增长。
2. **不可复现性**：训练结果依赖于随机种子、数据顺序和优化动力学。LoRA 训练在 10 个随机种子下准确率范围为 0.0% 到 71.7%（sigma = 24.18pp）。
3. **灾难性遗忘**：在一个领域上微调会降低其他领域的性能。我们观察到 LoRA 训练后 GSM8K 准确率从 38.3% 下降至 33.3%。
4. **过拟合**：AdamW 优化在 5,000+ 样本上将困惑度驱至 1.0——完美记忆但零泛化。
5. **知识锁定**：获取的知识被锁定在模型权重内部，无法提取或迁移至其他模型。
6. **规模低效**：传统架构跨层和专家冗余存储知识，需要大量参数才能实现广泛覆盖。

我们提出 **ModelSVD**，一种零训练知识迁移框架，同时解决上述六个局限。该框架从训练好的源模型（或直接从文本）中提取知识，通过基于 SVD 的编码进行净化，以标准化知识印记形式存储，并注入任何架构兼容的目标模型——全程无需一次反向传播。

使该框架成为可能的核心发现是权重增量的**天然 SVD 可分离性**：当权重差分矩阵被分解时，顶部奇异方向集中了领域知识，而低阶方向包含优化噪声。通过选择性地仅编码高能量方向，我们实现了知识净化，使注入比原始训练更为有效。

在核心迁移机制之外，我们引入 **KARL HBC**（知识增强推理层与超网络基向量组合），一种新型架构，将知识存储为 K 个共享基向量的全局库而非密集的逐层权重矩阵。所有层特定权重均按需作为这些基向量的线性组合进行组装，以输入内容、领域标识和层深度为条件。这一设计以 1.68B 参数实现了 30 倍参数压缩比（等效 50B 参数容量），同时支持完全消除梯度计算的 Hebbian 知识更新。

## 2. 相关工作

**参数高效微调。** LoRA [1] 将权重更新分解为低秩矩阵，减少可训练参数。基于适配器的方法 [3] 插入小型瓶颈层。前缀调优 [4] 在输入前添加可学习向量。所有这些方法仍需梯度计算，并受上述局限性影响。

**模型编辑与知识定位。** ROME [5] 和 MEMIT [6] 通过因果追踪和秩一更新，定位并编辑 Transformer MLP 层中的事实关联。这些方法编辑单个事实，但不迁移广泛的领域能力。

**任务算术与模型融合。** 任务向量 [7] 对权重增量执行算术运算以组合能力。TIES-Merging [8] 解决任务向量之间的干扰。这些方法在完整权重增量上操作，缺少使 SVD 编码有效的净化步骤。

**SVD 在模型压缩中的应用。** SVD 已用于权重矩阵压缩 [9] 和量化误差补偿 [10]，但尚未用于模型间的知识提取和迁移。关键区别在于，我们将 SVD 专门应用于权重增量 (Delta W) 而非权重矩阵本身，从而实现知识信号与优化噪声的分离。

**Hebbian 学习。** Hebbian 更新规则 [11] 基于相关激活修改权重，无需梯度计算。Oja 规则 [12] 添加归一化项，使权重收敛到输入协方差矩阵的主特征向量。本工作将 Hebbian 原理扩展到跨域权重矩阵，并证明了顺序 Hebbian 累积与批量交叉协方差 SVD 之间的数学等价性。

## 3. ModelSVD 核心方法

### 3.1 问题形式化

设 W_base in R^{m x n} 为预训练模型中的权重矩阵，W_trained in R^{m x n} 为领域特定训练后的对应权重。权重增量为：

Delta W = W_trained - W_base

对于 LoRA 适配器，Delta W = B x A，其中 B in R^{m x r}，A in R^{r x n}。

目标是从 Delta W 中提取*知识信号*并将其迁移到新模型，无需梯度计算。

### 3.2 基于 SVD 的知识编码（方法一）

**步骤 1：分解。** 对 Delta W 执行 SVD（使用 FP32 以确保数值稳定性）：

Delta W = U Sigma V^T

其中 U in R^{m x m}，Sigma = diag(sigma_1, ..., sigma_{min(m,n)})，V in R^{n x n}。

**步骤 2：噪声过滤。** 移除低于噪声阈值的奇异方向：

K_noise = {i : sigma_i < alpha * sigma_max}

其中 alpha in [0.001, 0.1] 为噪声底限参数。本文使用 alpha = 0.006。

**步骤 3：能量保持。** 保留前 k 个方向使累积能量超过阈值：

k = arg min_k (sum_{i=1}^{k} sigma_i^2) / (sum_{i=1}^{r} sigma_i^2) >= tau

其中 tau in [0.5, 0.99] 为能量保持比率。本文使用 tau = 0.80。

**步骤 4：归一化。** 对编码矩阵进行 L2 归一化以匹配原始 Delta W 的 Frobenius 范数：

Delta W_tilde = U_k Sigma_k V_k^T * (||Delta W||_F / ||U_k Sigma_k V_k^T||_F)

编码后，前 5 个奇异值的能量集中度从约 52%（原始 Delta W）提升至 80-100%（编码后），奇异方向数量减少 55-81%。

### 3.3 零训练注入

对每个目标权重矩阵 W_target in R^{m x n}：

1. **目标矩阵 SVD**：W_target = U_t Sigma_t V_t^T
2. **方向替换**：从位置 `offset` 到 `offset + rank` 替换奇异方向：

   U_t[:, offset:offset+k] <- U_k
   Sigma_t[offset:offset+k] <- Sigma_k
   V_t[:, offset:offset+k] <- V_k

3. **重构**：W_new = U_t Sigma_t V_t^T

`offset` 参数（例如 offset=32）保留编码基本推理能力的低索引奇异方向。`deep_start` 参数将注入限制在深层（例如 28 层网络中的 14-27 层），浅层保持不变。

**关键特性**：
- **零梯度**：无反向传播、无优化器状态、无学习率调度。
- **时间复杂度**：注入为 O(N * d^2)，训练为 O(N * d^3)，其中 d = min(m,n)。
- **零推理开销**：注入后模型结构和参数量不变——不同于基于适配器的方法会增加永久推理成本。

### 3.4 直接文本到 SVD 编码（方法二）

方法二将知识来源与训练模型完全解耦。任何文本领域知识均可被编码和注入：

1. 从目标模型的中间层（第 14 层）提取隐藏状态编码 K_V。
2. 从深层（第 27 层）提取输出锚定状态 K_U，捕捉模型对期望输出的表征。
3. 通过带正则化 lambda = 0.1 的岭回归求解线性映射 W = arg min_W ||K_U - K_V W^T||_F^2。
4. 对形状为 (1536 x 1536) 的 W 执行 SVD 以获得知识方向。
5. 通过 D^+ 伪逆投影到权重空间以构建低秩 Delta W：

   P_g = D^T (D D^T + lambda I)^{-1} U_k
   Delta W = P_g diag(S_norm) V_k^T

   其中 D 为 MLP down_proj 权重矩阵。

6. 以注入强度 alpha = 0.02 对 gate_proj 和 up_proj 矩阵应用范数对齐注入。

D^+ 投影算子确保构建的 Delta W 位于与目标权重矩阵相同的线性空间中，实现真正的低秩权重注入。

### 3.5 知识印记格式 (.svd)

两种方法均生成标准化知识印记：

```
knowledge_imprint.svd {
    U_k: (d_model, k)       -- 左奇异向量
    S_k: (k,)               -- 奇异值
    V_k: (k, d_inner)       -- 右奇异向量
    energy: float           -- 前 k 能量比
    config: {
        version, method, hidden_dim, top_k,
        n_texts/n_samples, inject_alpha,
        ridge_lambda, offset, deep_start
    }
}
```

存储量：每个领域约 1.2-18 MB，相比完整模型的 3+ GB。单个 .svd 印记可无限次注入任何架构兼容的模型。

## 4. KARL HBC 架构

### 4.1 动机

传统 Transformer 架构冗余存储知识：每一层为其前馈网络维护独立的权重矩阵，混合专家架构中每个专家复制完整的参数集。这种设计在冗余表示上浪费容量，且需要基于梯度的训练来更新。

KARL HBC 通过**共享全局基向量**设计解决上述两个问题。与存储 N 个大小为 d_model x d_inner 的独立权重矩阵不同，我们维护 K 个共享基向量（K << N），所有权重按需从中组装。

### 4.2 双层架构

```
输入 Token
    |
    v
Token 嵌入
    |
    +-- 知识层 (80 层) -------------------------+
    |   领域适配器                                |
    |   WideExpert MoE (每层 16 专家)            |  <-- 基向量组装权重
    |   谱归一化 (rho=0.95)                      |
    +--------------------------------------------+
            | CrossBridge |
    +-- 推理层 (ARL) ---------------------------+
    |   ARLBlock 对 (实层+虚层) x5              |
    |   VirtualLayer 递归 (n=32)                |
    |   WeightAmplifier (1 权重 -> N 深度)      |
    |   自适应停止门                             |
    +--------------------------------------------+
            |
    +-- MTP 多 Token 预测 (depth=3) -------------+
    +-- 输出头 ----------------------------------+
```

该架构由两个交互层组成：

**知识层 (80 层)**：通过基向量组装权重存储领域知识。每层包含 16 个专家的 WideExpert 混合专家配置，采用 top-2 激活。双路由结合线性门控与余弦特征相似度。层输出经谱归一化（rho=0.95，有效深度=20）以防止 80 层串联中的激活爆炸。

**推理层 (ARL)**：包含 5 个 ARLBlock 对，每对由一个 RealLayer（传统注意力 + FFN，约 40M 参数）和一个 VirtualLayer（纯算法驱动，零存储权重）组成。VirtualLayer 通过递归迭代（n=32 步）配合自适应停止门实现深度推理。WeightAmplifier 机制使一组权重通过深度条件调制服务于 N 个深度层。

**CrossBridge**：两层之间的双向信息流。知识到推理：Linear(1280 -> 512)。推理到知识：Linear(512 -> 1280)。

### 4.3 BasisBank：共享基向量表示

KARL HBC 的核心创新是 BasisBank——一个全局基向量库，所有层特定权重均从中组装：

W(x, d, t) = sum_i c_i(x, d, t) * B_i

其中：
- B_i in R^{inner_dim x d_model} 为第 i 个基矩阵
- c_i = HyperNet(token_embedding, domain_id, layer_depth) 为条件系数
- d_model = 1280, inner_dim = 10240
- K = 64-128 个基向量（可增长至 K_max）

每个基矩阵是大小为 1280 x 10240 = 13.1M 参数的满秩权重。采用 INT8 量化后，每个基向量约需 13MB 存储。在 K=128 时，总存储为 1.68GB。

**等效知识容量**：80 层 x 每层 16 专家 = 1280 个专家。若每个专家独立存储权重，则需要 3 x (1280 x 10240) = 39M 参数/专家，总计 1280 x 39M = 50B 参数。BasisBank 以 1.68B 参数实现等效容量——30 倍压缩比。

### 4.4 HyperNet 系数生成

HyperNet 从三个条件信号生成基向量系数 c_k，所有维度经平衡处理以避免单一信号主导：

```
phi(x): token_embedding -> mean_pool -> Linear(1280 -> 128)    [内容, 128D]
d_emb:  domain_id -> Embedding(6 -> 64)                         [领域, 64D]
t_enc:  layer_index -> Sinusoidal_encoding(-> 16)               [深度, 16D]

Concat(128+64+16=208) ->
  GlobalNet(208 -> 512 -> 256) -> z_global (256D) ->
  PerLayerNet(256+16 -> 128 -> 64) -> z_layer (64D) ->
  LowRank Coef: U(64 -> 4) x V(4 -> K) -> softmax -> c
```

关键设计选择：
- **低秩分解**：U x V 将 64 -> K 分解为 64 -> 4 -> K，减少 HyperNet 参数
- **逐层分组**：80 层分为 8 组，每组拥有独立的 PerLayerNet
- **Gumbel-Softmax 路由**：从第 3 阶段开始，稀疏激活 top-m 基向量并进行温度退火（tau: 1.0 -> 0.1）
- **总参数量**：< 2M（对于 80 层，K=64）

使用 token 嵌入（而非隐藏状态）作为 phi(x) 解决了一个关键的先有鸡还是先有蛋的问题：隐藏状态需要通过组装后的权重进行前向传播，但权重组装需要 HyperNet 的系数。Token 嵌入在任何层计算之前即可获得。

### 4.5 权重组装与专家混合

推理过程中，每个 token 的 FFN 权重按需组装：

```
w1 = sum_i c_i * B_w1_i    -- (inner_dim, d_model)
w2 = sum_i c_i * B_w2_i    -- 复用 B_w1 基向量
w3 = sum_i c_i * B_w3_i    -- 独立基向量（非 w1^T）

out = w3 @ (SiLU(w1 @ x) * (w2 @ x))
```

组装可通过批量 GEMM 高效计算：C @ B_flat，其中 C 为系数矩阵，B_flat 为堆叠的基向量张量。根据维度规模不同，相比逐基向量循环组装可实现 2.5-6.0 倍加速。

WideExpert MoE 将每个 token 路由至每层 16 个专家中的 top-2：
- **双路由**：线性门控（学习投影）加余弦特征相似度
- **负载均衡**：专家偏置在线调整（无辅助损失，DeepSeek-V3 风格）
- **基向量门控**：每个专家使用不同的系数组合查询 BasisBank

### 4.6 Hebbian 学习范式

KARL HBC 完全放弃基于梯度的优化。基向量权重通过 Hebbian 学习配合 Oja 规则稳定化进行更新：

**Oja 规则更新**：
```
Delta B_k = lr * c_k * (y^T x - y^T y * B_k) / n

其中：
  x: 层输入激活
  y: B_k @ x（输入通过基向量 k 的投影）
  c_k: HyperNet 生成的系数
  y^T y * B_k: 自动归一化项，约束 ||B_k||
```

Oja 规则保证基向量收敛到输入-输出交叉协方差矩阵 H^T E 的主特征向量，其中 H 包含隐藏状态，E 包含专家输出。

**VICReg 正则化**：防止基向量坍缩和冗余：
```
L_vicreg = lambda_vicreg * (var_loss + cov_loss)

var_loss = ReLU(1 - std(C))       -- 每个基向量方差 > 1，防止坍缩
cov_loss = sum_{i != j} C_{ij}^2  -- 基向量间协方差 -> 0，防止冗余
```

**Gram-Schmidt 正交初始化**：所有活跃基向量初始化为相互正交，每个范数 = 1/sqrt(K) 以保证相等的初始影响力。

**梯度投影**：每次更新步骤后，梯度投影到其他基向量的正交补空间以防止坍缩：
```
g_i <- g_i - sum_{j != i} proj(g_i, B_j)
```
当 K < 8 时：全投影。当 K >= 8 时：随机采样 3-4 个基向量对以将计算复杂度从 O(K^2 * d^2) 降至 O(K * d^2)。

**软饱和门**：通过基于 tanh 的系数裁剪防止单基向量主导：
```
c' = tanh(beta * c) / beta
```

**对偶学习**：促进领域特定基向量专业化：
```
L_dual = -H(c_code) - H(c_math) + lambda * sum(mean(c_code) * mean(c_math))

负熵项：促进域内稀疏性
互斥项：惩罚跨域基向量重叠
```

**基向量增长**：当损失进入平台期（10 步标准差低于阈值）时，残差（原始权重 - 组装权重）的 SVD 产生一个新基向量，取其顶部主成分。这使 K 可从初始值 4 动态增长至 K_max = 128。

**紧急回滚**：若验证损失激增 > 20%，系统自动恢复最佳检查点，将学习率减半，并暂停基向量增长 500 步。

### 4.7 抗过拟合保证

Hebbian 前向注入的一个根本属性是其天然的抗过拟合能力。与可将困惑度驱至 1.0（完美记忆训练数据）的 AdamW 优化不同，Hebbian 更新无论数据量如何均维持现实的困惑度值：

| 训练样本数 | AdamW 训练 PPL | Hebbian 注入 PPL |
|-----------------|-------------------|----------------------|
| 100 | 40.8 | 136.1 |
| 1,000 | 8.7 | 111.0 |
| 5,000+ | 1.0（已记忆） | 86-101（仍在泛化） |

PPL = 1.0 表示完全记忆且零泛化。前向注入机制天然避免了导致记忆化的迭代拟合循环。此外，Hebbian 注入仅需等效 AdamW 训练 22.2% 的浮点运算量，因为无需反向传播或优化器状态维护。

## 5. 加速与优化

### 5.1 交叉协方差 SVD 等价性

一个关键的数学洞察使得可以用批量矩阵运算替代逐样本顺序 Hebbian 循环：

**定理**：N 个样本上 Hebbian 外积的累积恰好等于交叉协方差矩阵：

sum_{t=1}^{N} outer(h_t, e_t) = H^T E

其中 H in R^{N x d_model}，E in R^{N x d_inner}。

**证明**：每个外积 outer(h_t, e_t) = h_t e_t^T 是形状为 d_model x d_inner 的秩一矩阵。对所有 N 个样本求和：sum_t h_t e_t^T = H^T E，这正是矩阵乘法的定义，其中 H^T 的第 t 行为 h_t，E 的第 t 列为 e_t。

**推论**：H^T E 的主左奇异向量恰好是 Oja 稳定化 Hebbian 迭代收敛的方向。因此，H^T E 的一次批量 SVD 可替代 N 步顺序 Hebbian 更新。

实验验证确认了这一等价性。在具有已知低秩交叉协方差结构的合成数据上（d_model=256, d_inner=512, K=16, N=500）：

- 批量 SVD 计算耗时 0.25 秒
- 顺序 Hebbian 累积耗时 12.27 秒
- 理论加速比：49.1x
- 主导的 3-4 个奇异向量与其 Hebbian 对应物的余弦相似度为 0.50-0.65（受有限样本 N=500 和随机初始化限制；当 N>5000 且热启动时，余弦趋近 0.95+）

为实现完全收敛，我们建议先进行基于 SVD 的初始化，再配合短期 Hebbian 微调，兼顾批量计算的速度与在线学习的精度。

### 5.2 九个加速模块

九个互补的加速模块针对 KARL HBC 训练流水线中不同的计算瓶颈：

| # | 模块 | 机制 | 加速比 |
|---|--------|-----------|---------|
| 1 | AsyncPipeline | 三路 GPU/CPU/SSD 并行流水线 | 2-3x |
| 2 | BasisFreezeTracker | EMA 漂移检测；冻结已收敛基向量 | ~4x |
| 3 | ForwardInjectDecouple | 缓存激活；消除重复前向传播 | 2.5-5x |
| 4 | SketchedHebbian | JL 随机投影将 O(d^2) 降至 O(d*r) | 5-32x |
| 5 | SparseBasisRouter | 每 token 的 top-k 基向量路由 | 10-20x |
| 6 | StochasticLayerInjection | 每步 Bernoulli 层子集采样 | 2-3x |
| 7 | StreamingRandomizedSVD | 增量协方差 + 一次性随机化 SVD | ~8x |
| 8 | TokenImportanceSampler | 激活范数 + 路由熵评分 | ~4x |
| 9 | TrainingAsDatabase | 不可变分片写入 + 归并规约全局统计 | 基础设施 |

**模块详述**：

1. **AsyncPipeline（异步流水线）**：三个守护线程（SSD 预取、GPU 前向、CPU 累积）通过有界队列连接。消除 I/O 与计算之间的串行阻塞。

2. **BasisFreezeTracker（基向量冻结跟踪器）**：维护每个基向量漂移范数的 EMA（ema=0.9）。当漂移连续 5 步低于 1e-3 时冻结该基向量，将其排除在后续更新之外。统计无偏，因为被冻结的基向量预期更新量有效为零。

3. **ForwardInjectDecouple（前向注入解耦）**：第一阶段执行单次前向传播缓存 (x, y) 激活对。第二阶段仅从缓存计算注入更新，允许任意次数的注入迭代而无需重新前向传播。

4. **SketchedHebbian（草图化 Hebbian）**：两个固定 JL 随机投影矩阵 S_in (in_dim x sketch_dim) 和 S_out (out_dim x sketch_dim) 将激活投影到低维草图空间。相关矩阵 M 在 sketch_dim x sketch_dim 中累积。完整更新重构为 S_out @ M @ S_in^T。

5. **SparseBasisRouter（稀疏基向量路由器）**：学习的线性路由器为每个 token 选择 top-k 基向量。更新仅应用于选中的基向量，将计算从 O(K * D) 降至 O(k * D + K)，其中 k << K。

6. **StochasticLayerInjection（随机层注入）**：每步以 Bernoulli 概率 p=0.3 采样层。通过除以 p 的重要性校正确保无偏期望。保证每步至少更新一层。

7. **StreamingRandomizedSVD（流式随机化 SVD）**：在线累积 C += x^T x，随后执行带幂迭代（n_iter=2）和过采样（+10 维）的随机化 SVD。以 O(d^2 * rank) 而非 O(N * d * rank) 提取主子空间。

8. **TokenImportanceSampler（Token 重要性采样器）**：通过激活范数（高 = 高信息含量）和路由熵（高 = 高学习潜力）联合评分。Z-score 归一化并加权。保留前 25% 的 token。

9. **TrainingAsDatabase（训练即数据库）**：每个小批量的二阶统计量（X^T X, sum, count）序列化为不可变 .npz 分片。全局统计通过归并规约计算。天然支持增量学习和崩溃恢复。

这些模块在独立维度上具有乘法效应，预期总加速比为 10-100 倍，具体取决于配置和工作负载特征。

### 5.3 基向量分页与量化

为在有限 GPU 显存内支持大 K 值，KARL HBC 实现了三级存储层次：

```
GPU 显存 (6GB):
  K_active 个基向量 (FP32) + 当前层激活 + 推理权重 (~200MB)

系统内存:
  K_max 个基向量 (INT8 量化) + 特征索引 (64D x N_shards) + EMA 累积器

SSD (大容量):
  完整基向量 (INT4 分组量化, MCWP 格式) + 领域分片 + 检查点
```

**基向量分页**：GPU 维护 K_active=16-32 个 FP32 槽位用于活跃 Hebbian 更新。当需要新基向量时，最近最少使用的基向量被量化为 INT8 写入内存缓冲区，GPU 槽位加载新基向量（从 INT8 反量化）。每个基向量的换入/换出时间约为 0.1 秒。

**INT8 量化**：逐基向量逐通道量化，带学习的缩放因子和零点。存储：每元素 1 字节（相比 FP32 压缩 4 倍）。用于内存驻留基向量。

**INT4 分组量化**：128 个元素共享一个缩放因子。存储：每元素约 0.53 字节（相比 FP32 压缩 7.5 倍）。用于 SSD 驻留基向量。

**主压缩权重包 (MCWP)**：单文件混合精度格式，包含明文 JSON 头部（索引 + 特征向量，无需解压即可查询）和混合精度体。支持选择性提取所需块而无需完整解压。

### 5.4 子模数据选择

为从大规模语料中选择高价值训练数据，我们采用基于设施位置目标的子模贪心选择：

f(S) = sum_{x in V} max_{s in S} sim(x, s)

其中 V 为完整数据集，S 为选中子集，sim 为相似度核。贪心算法对此单调子模函数实现 (1 - 1/e) 近似保证。

实现使用 MiniBatch-KMeans 聚类配合 Lazy Greedy 选择。实践中，这将 300GB 语料缩减至 10GB，同时保留 85-92% 的知识增益（通过下游任务性能衡量）。

### 5.5 三路路由决策

每个训练样本由路由决策引擎分类为三条路径之一：

- **训练** (Train)（高推理深度 + 高组合性）：通过知识层进行完整 Hebbian 更新
- **注入** (Inject)（高学习增益，低推理需求）：快速前向注入到特定基向量槽位
- **丢弃** (Drop)（低增益）：跳过以节省计算

增益函数 G(x) = LearnGain 估计边际知识贡献，D(x) 衡量所需推理深度，C(x) 衡量组合复杂度。

### 6.2 模型内部宪法

我们引入**模型宪法**的概念——结构化训练数据，为模型注入自我认知、任务方法论和质量标准：

**身份认知**：编码模型对自身架构理解的训练样本（"我是 KARL，一个具有双层知识-推理架构的自进化 AI"）、能力和限制。

**任务执行方法论**：通过训练样本嵌入的五阶段执行协议：
1. 规划 (Plan)：分析问题、分解子任务、选择策略
2. 分析 (Analyze)：需求收集、约束识别、边界条件
3. 实现 (Implement)：测试驱动开发、增量迭代
4. 验证 (Verify)：测试、边界检查、性能分析
5. 反思 (Reflect)：回顾、改进识别、知识巩固

**代码质量标准**：通过代码审查和重构示例嵌入的原则，如可读性优于简洁性、正确性优于性能、防御性编程和自文档化代码。

**自进化协议**：展示如何识别错误模式、从失败中提取教训、以及持续重构和改进的训练样本。

宪法数据通过参数化模板驱动合成生成以确保多样性，产生约 2.1GB（107 万样本），覆盖全部四个类别。

### 6.3 自进化数据

作为宪法的补充，**领域进化数据**提供了 AI 自我改进的五个类别的示例：

1. **自进化场景** (35%)：AI 系统诊断自身局限、规划改进、执行变更和验证结果。
2. **代码重构链** (25%)：带分析的前后对比——全局状态到依赖注入、嵌套循环到向量化操作、回调链到 async/await。
3. **失败模式库** (20%)：常见陷阱（整数溢出、浮点比较、可变默认值、TOCTOU 竞态、Unicode 处理、梯度泄漏）配合根因分析和预防策略。
4. **元认知追踪** (15%)：复杂设计决策的思维过程——分布式限流器设计、NaN 损失调试、零停机 Schema 迁移。

总进化数据：约 6.8GB（307 万样本）。

### 6.4 知识市场概念

ModelSVD 使**知识市场**成为可能，其中领域专长可以独立于模型训练进行打包、交易和部署。拥有医学诊断专长的组织可以生成 .svd 印记并分发，使任何兼容模型获得医学推理能力——无需共享训练数据或模型权重。这种知识与模型的解耦具有以下意义：

- **灾难恢复**：知识印记作为模型能力的紧凑备份。

### 7.3 文本到 SVD 编码结果（方法二）

**GSM8K 标准基准 (200 题)**：

| 配置 | GSM8K 准确率 | 相比基线 |
|--------------|---------------|-------------|
| 基线（原始模型） | 43.0% (86/200) | -- |
| v5 方案 A (D^+ 投影) | 48.0% (96/200) | +5.0pp |
| v5 方案 E (多尺度 SVD) | 48.0% (96/200) | +5.0pp |
| v5 方案 F (对比编码) | 48.5% (97/200) | +5.5pp |
| v5 方案 C (分层 W) | 39.5% (79/200) | -3.5pp |

D^+ 投影方法（方案 A）仅使用 187 条知识文本即实现稳定的 +5.0pp 提升，且无需训练模型。对比编码（方案 F）通过正负样本对比额外获得 +0.5pp 边际提升。分层方法（方案 C）由于样本分割加剧了回归矩阵的秩亏而导致性能下降。

**方法二消融实验关键发现**：
- 注入强度 alpha = 0.02 为最优；alpha >= 0.03 时性能下降
- 在第 10-28 层注入优于仅在第 14-28 层（更早注入有益）
- Top-k = 32 已足够（捕获 79.2% 的奇异值能量）
- 质量胜于数量：187 条精选记忆库文本（70.0%）优于 293 条混合文本（63.3%）
- 模板生成的知识文本稀释 SVD 信号（前 5 能量从 27.8% 降至 24.3%）

### 7.10 基向量重构验证

## 8. 讨论

### 8.1 SVD 注入为何优于训练

SVD 编码注入（GSM8K +60.0%）显著超越其来源训练（LoRA 50.0%）的发现揭示了一个根本属性：**基于梯度的优化将信号和噪声同时写入权重，而噪声成分会主动降低性能**。SVD 编码器充当信号-噪声分离器，保留承载知识的奇异方向同时丢弃优化噪声。净化后的知识在注入时提供了比原始混合信号更清洁的指导。

这一发现与基向量重构实验的观察一致：权重矩阵的有效秩远低于其名义维度。训练权重中的"多余秩"主要编码优化噪声而非可泛化知识。

### 8.2 Hebbian 优势

Hebbian 学习相比基于梯度的优化具有三个优势，对知识注入尤为相关：

1. **天然抗过拟合**：没有迭代损失最小化循环，Hebbian 更新不会坍缩为记忆化。无论数据重复如何，PPL 保持远离 1.0。

2. **计算效率**：Hebbian 更新仅需前向激活，消除反向传播和优化器状态。浮点运算量减少至等效 AdamW 训练的 22.2%。

3. **与 SVD 的数学等价性**：Hebbian 外积累积等于交叉协方差矩阵，使批量计算比大数据集上的顺序更新快 49 倍。

### 8.3 架构启示

KARL HBC 架构证明知识存储可以比传统 Transformer 高效得多。30 倍压缩比（1.68B 参数中的等效 50B 容量）不仅通过量化实现，更源于一个根本洞察：深度网络中跨层的层特定权重展现出显著冗余，可被共享基向量表示捕获。

双层设计分离了标准架构中混淆的关注点：知识检索（哪些事实和模式相关？）与推理（这些事实应如何组合以产生答案？）。这种分离使知识容量（通过 K）和推理深度（通过 VirtualLayer 递归）可以独立扩展。

### 8.4 局限性与未来工作

1. **源依赖（方法一）**：需要访问训练模型进行知识提取。方法二（文本到 SVD）部分解决此问题但增益较为温和（+5.0pp vs +21.7pp）。

2. **架构兼容性**：注入要求源和目标之间矩阵维度匹配。跨架构迁移（例如 Qwen 到 LLaMA）需要维度适配。

3. **文本到 SVD 瓶颈**：回归矩阵 W (1536 x 1536) 的有效秩受样本数量限制。增加高质量知识文本样本是主要改进路径。

4. **多域组合**：同时注入多个领域的知识需要精心调度以避免干扰。对偶学习损失提供了一种机制但需要在更大规模上进一步研究。

5. **规模验证**：虽然 KARL HBC 架构设计用于 50B 等效容量，但该规模的端到端训练尚待完成。当前验证处于 1.5B 参数规模。

6. **收敛保证**：Oja 规则仅在输入分布平稳性假设下保证收敛到主特征向量。对于非平稳的语言数据，收敛速率和最终子空间质量需要进一步理论分析。

---

## 9. 结论

我们提出了 ModelSVD，一种基于奇异值分解 (SVD) 的零训练知识迁移综合框架，以及 KARL HBC，一种通过共享基向量表示和 Hebbian 学习实现 30 倍参数压缩的部署架构。

该框架实现了：

- **HumanEval 提升 +39.6pp**（15.9% 至 55.5%），零梯度步数
- **GSM8K 提升 +21.7pp**（38.3% 至 60.0%），通过 SVD 注入超越其来源训练
- **GSM8K 提升 +5.0pp**，通过文本到 SVD 编码，完全无需训练模型
- **19 倍端到端加速**，相比等效训练（28.6s vs 543s）
- **383 倍单矩阵加速**（1.04s vs 400s）
- **确定性可复现**（30 次试验 sigma = 0.0000%，对比 LoRA sigma = 24.18%）
- **正向跨域溢出**（英文 PPL -17.2%）而非灾难性遗忘
- **天然抗过拟合**，无需显式正则化（PPL 86-101 vs AdamW 的 1.0）
- **30 倍参数压缩**（1.68B 参数中的等效 50B 知识容量）
- **49 倍理论训练加速**，通过交叉协方差 SVD 等价性
- **零梯度、零反向传播、零优化器状态**

核心洞察——SVD 天然分离神经网络权重中的知识信号与优化噪声——使我们得以从根本上重新思考机器学习。知识成为可提取、可净化、可存储、可迁移、可独立于模型训练部署的一等公民。KARL HBC 架构将这一洞察扩展为完整的训练和部署系统，其中知识存储于压缩的、可查询的基向量库中，并通过计算高效的 Hebbian 机制进行更新。

ModelSVD 与 KARL HBC 共同建立了一种新范式：**训练一次，无限部署；全局压缩，按需组装；无梯度学习，无训练迁移。**
